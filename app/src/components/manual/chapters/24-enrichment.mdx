Enrichment fetches extra information about each location's panorama and stores it on the location. This metadata can store **non-boolean data** (numbers, dates, strings), unlike tags, which makes it the foundation for filtering, sorting, and analysis.

## Available fields

Five fields are **on by default**:

- **Altitude.** Elevation of the pano in meters.
- **Country code.** ISO country of the coverage.
- **Camera type.** The generation/quality: gen1, gen2, gen4, tripod, trekker, or badcam.
- **Pano type.** Official, unknown, or user-uploaded.
- **Image date.** The year-month of capture.

Five more are **off by default** and must be opted into:

- **Exact date.** The precise capture timestamp, resolved by binary-searching Google's servers. Much slower than other fields (multiple requests per location). Display format (date only vs date + time) and timezone (location-local vs UTC) are configurable in **Settings → Street View → Date Picker**.
- **Timezone.** The pano's IANA timezone (e.g. `America/New_York`). Pairs with exact date for time-of-day filtering in local time.
- **Driving direction.** Which side of the road the coverage car was driving on.
- **Uploader.** The name of the user or entity that uploaded the panorama.
- **Coverage dates.** The full list of capture dates available at that location across all time periods.

Plugins can register additional enrichment fields.

## Choosing which fields to enrich

Click the **gear icon** next to **Enrich locations with metadata** in the map settings dropdown to open the field picker. Each field has a checkbox; toggling one off means it will be skipped during enrichment. This is a per-map setting.

## Running enrichment

- **Automatically on view.** Enable **Enrich locations with metadata** in the map settings dropdown. Each location is enriched as you open it in the viewer. Metadata is always fetched for display; this setting controls whether it gets **saved** to the location. Disable it if you want to browse without modifying location data.
- **In bulk.** Run **Enrich metadata fields** from the command palette, choosing a scope. The runner skips locations that are already enriched unless you force a refresh. Slow fields (exact date) show their own progress.

## Using enriched data

Once locations carry metadata, it unlocks several features:

- **Filter by metadata.** Build selections based on field values - for example, all locations from 2023, or all gen4 coverage above 500m altitude, etc. See <ChapterLink id="filters" />.
- **Apply metadata as tags.** Pick a metadata field and <App /> creates one tag per distinct value, assigning each location automatically. For example, applying **countryCode** tags your entire map by country in one step. Date fields offer grouping choices (year, year-month, day, month of year, hour of day). Number fields can be bucketed by a fixed width (e.g. altitude in 500m bands). This is in the command palette under **Tags**. See <ChapterLink id="tags" />.
- **Gradient coloring.** The Gradient plugin colors locations on a spectrum by any numeric or date field. See <ChapterLink id="builtin-plugins" />.
- **Manage fields.** Rename, retype, merge, or delete metadata fields from the map settings dropdown. See <ChapterLink id="managing-fields" />.
